Consolidamento degli IF annidati

Lezione di laboratorio con Flowgorithm

Obiettivi della lezione

  • Applicare correttamente gli IF annidati in semplici problemi.
  • Riconoscere gli errori più frequenti nelle condizioni.
  • Usare Flowgorithm per costruire e testare algoritmi.
  • Controllare il comportamento dell’algoritmo con input diversi.

Cosa faremo oggi

  1. Ripasso rapido del significato di IF annidato.
  2. Esercizi guidati in Flowgorithm.
  3. Correzione di algoritmi con errori.
  4. Attività finale autonoma.

Richiamo rapido

Un IF annidato si usa quando una seconda decisione va presa solo in uno dei casi già selezionati.

Esempio di idea logica:

  • prima controllo se il numero è positivo, negativo oppure zero;
  • solo se è positivo, faccio un secondo controllo.
Attenzione

Non basta che l’algoritmo sembri corretto. Va sempre provato con valori diversi, soprattutto con i casi limite.

Esercizi guidati con Flowgorithm

Per ogni esercizio segui sempre questa procedura:

  1. Leggi con attenzione la consegna.
  2. Crea il diagramma in Flowgorithm.
  3. Esegui il programma con almeno 3 input diversi.
  4. Controlla se l’output è corretto in tutti i casi.

Esercizio 1 — Numero positivo e grande

Leggere un numero intero e:

  • dire se è positivo, negativo oppure zero;
  • se è positivo, dire anche se è maggiore di 100 oppure minore o uguale a 100.

Esercizio 2 — Età e categoria

Leggere l’età di una persona e stampare:

  • bambino se età < 12
  • ragazzo se età è compresa tra 12 e 17
  • adulto se età ≥ 18

In più, se la persona è adulta, scrivere anche: può votare.

Esercizio 3 — Pari, dispari e segno

Leggere un numero intero e stampare:

  • pari positivo
  • pari negativo
  • dispari positivo
  • dispari negativo
  • zero
Suggerimento

Spesso conviene controllare prima il segno del numero e poi, dentro il caso corretto, controllare se è pari o dispari.

Errori tipici da riconoscere

Errore 1 — Condizioni nell’ordine sbagliato

SE voto >= 6 ALLORA
    scrivi "Sufficiente o più"
ALTRIMENTI SE voto >= 8 ALLORA
    scrivi "Ottimo"
FINE SE

Problema: il controllo voto >= 8 non verrà mai raggiunto, perché 8 soddisfa già la prima condizione.

Errore 2 — IF separati invece di una struttura corretta

SE numero > 0 ALLORA
    scrivi "Positivo"
FINE SE

SE numero mod 2 = 0 ALLORA
    scrivi "Pari"
FINE SE

Problema: l’algoritmo stampa messaggi separati, ma non distingue davvero tutti i casi richiesti dalla consegna.

Errore 3 — Dimenticare il caso zero

SE numero > 0 ALLORA
    scrivi "Positivo"
ALTRIMENTI
    scrivi "Negativo"
FINE SE

Problema: lo zero non è né positivo né negativo.

Attività di debug

Osserva gli algoritmi seguenti. Per ciascuno devi:

  1. dire che cosa non funziona;
  2. indicare un input che mostra l’errore;
  3. correggere il diagramma in Flowgorithm.

Debug 1

Leggi eta
SE eta >= 18 ALLORA
    Scrivi "Adulto"
ALTRIMENTI SE eta >= 12 ALLORA
    Scrivi "Bambino"
ALTRIMENTI
    Scrivi "Ragazzo"
FINE SE

Debug 2

Leggi numero
SE numero > 0 ALLORA
    SE numero > 100 ALLORA
        Scrivi "Positivo e grande"
    FINE SE
ALTRIMENTI
    Scrivi "Non positivo"
FINE SE

Domanda: che cosa succede se inserisco 25?

Debug 3

Leggi numero
SE numero mod 2 = 0 ALLORA
    Scrivi "Pari"
ALTRIMENTI
    Scrivi "Dispari"
FINE SE

SE numero < 0 ALLORA
    Scrivi "Negativo"
FINE SE

Domanda: questo algoritmo distingue davvero i casi pari positivo, pari negativo, dispari positivo, dispari negativo e zero?

Procedura consigliata in Flowgorithm

  1. Apri un nuovo file.
  2. Inserisci le variabili necessarie, per esempio numero oppure eta.
  3. Aggiungi un blocco Input.
  4. Inserisci il primo blocco Decision.
  5. Se serve, dentro un ramo inserisci una seconda decisione.
  6. Esegui l’algoritmo con più prove.
  7. Salva il file con un nome chiaro.

Valori da provare sempre

Un valore positivo, un valore negativo, zero e almeno un valore di confine, per esempio 12, 18 oppure 100.

I valori di confine aiutano a trovare errori nascosti nelle condizioni.

Attività finale autonoma

Consegna: realizzare in Flowgorithm un algoritmo che legga un numero intero e:

  • dica se è positivo, negativo oppure zero;
  • se è positivo, dica anche se è pari o dispari;
  • se è positivo, dica anche se è maggiore di 100 oppure minore o uguale a 100.

Prova l’algoritmo almeno con questi input:

-8, -3, 0, 2, 11, 100, 101

Checklist finale

  • Hai gestito tutti i casi richiesti?
  • Hai usato gli IF annidati solo dove servono?
  • Hai provato più input diversi?
  • Hai controllato anche i casi limite?
  • L’output è chiaro e leggibile?
